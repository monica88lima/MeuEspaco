  • 1 - DEFINA O QUE É CLASSE E O QUE É OBJETO E COMO PODEMOS CRIÁ-LOS NA LINGUAGEM C#

    Em POO a objetos, uma classe é um modelo ou uma estrutura que define um conjunto de propriedades e métodos que um objeto pode ter. Ela é uma espécie de "planta" para a criação de objetos. A classe define as características e comportamentos que os objetos daquela classe terão. Um objeto, por sua vez, é uma instância de uma classe. Ele representa um item ou entidade específica que possui os atributos e comportamentos definidos pela classe. Em outras palavras, um objeto é a materialização concreta de uma classe. Na linguagem c# para criar a instancia da classe usamos a palavra new

  • 2- COMO DEFINIR O CONCEITO DE ABSTRAÇÃO USADO NO PARADIGMA DA POO C#

    No contexto da programação orientada a objetos (POO), a abstração é um conceito fundamental que permite representar objetos do mundo real em um programa de forma simplificada, focando apenas nos aspectos relevantes para o sistema em questão. Ela envolve a criação de classes e objetos que encapsulam características e comportamentos específicos, abstraindo detalhes complexos para tornar o sistema mais compreensível e modular.

    A abstração na POO ajuda a simplificar o design do sistema, tornando-o mais modular, flexível e fácil de entender. Isso permite que os desenvolvedores se concentrem nos aspectos essenciais de um objeto, ignorando detalhes irrelevantes ou complexos. Isso é feito por meio da seleção cuidadosa das características e comportamentos que serão representados na classe, enquanto oculta os detalhes internos de implementação.

  • 4-O QUE É ENCAPSULAMENO E QUAIS SAO OS SEUS BENEFICIOS

    Encapsulamento é um conceito chave na programação orientada a objetos (POO) que envolve o agrupamento de dados (propriedades) e comportamentos (métodos) relacionados em uma única unidade chamada classe. O objetivo do encapsulamento é ocultar os detalhes internos de implementação de uma classe e fornecer uma interface externa consistente para interagir com o objeto.

    1. Ocultação de detalhes internos: O encapsulamento permite que os detalhes internos da implementação de uma classe sejam ocultados. Isso significa que outros componentes ou objetos externos não precisam conhecer ou depender de como uma determinada classe realiza suas operações internas. Isso facilita a modificação da implementação interna sem afetar outros componentes do sistema.

    2. Proteção de dados: Através do encapsulamento, é possível definir a visibilidade das propriedades e métodos de uma classe. Propriedades podem ser definidas como públicas, privadas ou protegidas, determinando quem tem acesso direto a elas. Isso ajuda a proteger os dados internos de uma classe, permitindo que sejam acessados e modificados apenas de acordo com as regras definidas pela classe.

    3. Modificação controlada: O encapsulamento também permite que a implementação interna de uma classe seja modificada de forma controlada. Ao encapsular os detalhes internos, é possível fornecer métodos públicos ou interfaces bem definidas para interagir com o objeto. Isso significa que, mesmo que a implementação interna da classe seja alterada, a interface externa permanece a mesma, o que minimiza o impacto em outros componentes do sistema.

    4. Reutilização de código: O encapsulamento promove a reutilização de código ao permitir que objetos de uma classe sejam usados em diferentes partes do sistema. Ao fornecer uma interface consistente e bem definida, os objetos encapsulados podem ser facilmente utilizados por outros componentes sem precisar conhecer os detalhes internos da classe.

    5. Modularidade e organização: O encapsulamento contribui para a modularidade do código, pois permite que as funcionalidades relacionadas sejam agrupadas em classes. Isso facilita a organização do código em unidades lógicas, o que torna o sistema mais compreensível, facilitando a manutenção e o desenvolvimento colaborativo.

  • 5-QUANDO UM CONTRUTOR DE UMA CLASSE É CHAMADO

    Um construtor de uma classe é chamado quando um objeto dessa classe é criado. Ele é responsável por inicializar o objeto, definindo seus valores iniciais e executando qualquer lógica necessária antes de o objeto ser utilizado.O construtor de uma classe é identificado pelo nome da classe e não possui um tipo de retorno explicitamente declarado. Ele pode receber parâmetros, que são utilizados para fornecer informações necessárias para a inicialização do objeto.

  • 6-SE VC CRIAR 5 OBJETOS DE UMA CLASSE, QUANTAS VEZES O CONSTRUTOR SERÁ CHAMADO

    Se você criar 5 objetos de uma classe, o construtor será chamado 5 vezes, uma vez para cada objeto criado. Cada objeto precisa passar pelo processo de inicialização, e isso é feito chamando o construtor da classe. Portanto, quando você usa o operador new para criar um objeto, o construtor correspondente é chamado para aquele objeto específico.

  • 7-POR QUE VOCÊ USA UM CONSTRUTOR?

    1. Inicialização de objetos: O construtor é usado para inicializar o estado de um objeto quando ele é criado. Isso envolve definir os valores iniciais das propriedades e estabelecer o estado inicial do objeto. O construtor garante que o objeto comece em um estado válido e consistente.

    2. Configuração de propriedades: O construtor permite a configuração inicial das propriedades de um objeto. Você pode passar argumentos para o construtor para definir os valores das propriedades no momento da criação do objeto. Isso permite uma personalização do objeto com base nas necessidades do programador.

    3. Execução de lógica inicial: O construtor pode ser usado para executar lógica inicial ou preparatória que deve ser realizada assim que o objeto for criado. Isso pode incluir a alocação de recursos, a abertura de conexões com bancos de dados ou a realização de outras tarefas necessárias antes que o objeto esteja pronto para uso.

    4. Controle de acesso: O construtor pode ser usado para definir a visibilidade e o acesso aos membros de uma classe. Você pode usar modificadores de acesso, como public, private ou protected, para controlar quem pode criar instâncias da classe e quais membros são acessíveis externamente.

    5. Sobrecarga de construtores: Em uma classe, você pode ter vários construtores com diferentes assinaturas (parâmetros). Isso permite a criação de objetos usando construtores diferentes, dependendo das necessidades específicas do momento.
      A sobrecarga de construtores fornece flexibilidade na criação e inicialização de objetos.

    Ou seja, o construtor é usado para inicializar objetos, configurar propriedades, executar lógica inicial, controlar acesso e fornecer flexibilidade na criação de objetos.
    Ele desempenha um papel fundamental na criação de instâncias de uma classe e garante que os objetos sejam devidamente configurados e prontos para uso.

  • 10- QUAL O CONCEITO DE HERANÇA? DE EXEMPLOS

    Herança é um conceito fundamental na programação orientada a objetos (POO) que permite criar novas classes a partir de classes existentes. É uma relação entre classes em que uma classe derivada herda os membros (propriedades e métodos) de uma classe base, adicionando ou modificando seu comportamento conforme necessário. A classe derivada, também chamada de subclasse ou classe filha, herda todas as características da classe base, também conhecida como superclasse ou classe pai. Isso significa que a classe derivada pode reutilizar o código da classe base, estendendo-o ou personalizando-o para atender às suas necessidades específicas. Um exemplo simples de herança pode ser visto no contexto de uma hierarquia de classes relacionadas a animais. Podemos ter uma classe base chamada "Animal" e classes derivadas como "Cachorro", "Gato" e "Pássaro". Veja o exemplo em C#:



    Nesse exemplo, a classe "Animal" é a classe base que define a propriedade "Nome" e o método "EmitirSom". As classes derivadas "Cachorro", "Gato" e "Passaro" herdam essas características da classe "Animal" e também têm comportamentos específicos adicionais. A classe "Cachorro" herda a propriedade "Nome" e o método "EmitirSom" da classe "Animal" e adiciona um novo método "Latir". A classe "Gato" herda da classe "Animal" e adiciona o método "Miar". A classe "Passaro" também herda da classe "Animal" e adiciona o método "Voar". Essa hierarquia de classes permite que você crie objetos de diferentes tipos de animais e acesse os membros herdados da classe base, além dos membros específicos de cada classe derivada. Também podemos acessar os membros específicos de cada classe derivada, como "Latir" para o cachorro, "Miar" para o gato e "Voar" para o pássaro.
    A herança permite a reutilização de código, promove a organização hierárquica de classes e facilita a criação de hierarquias mais complexas. Ela é um conceito poderoso na POO para modelar relações entre classes e promover a reutilização e extensibilidade do código.
